# OLED 1.5" Simulator

Симулятор дисплея **SSD1327** (128×128, 4-bit grayscale) для Raspberry Pi Pico.  
Дозволяє тестувати MicroPython-код на звичайному ПК без підключення залізничного обладнання.

---

## Корисні покликання

```
Робота з буфером: https://docs.micropython.org/en/latest/library/framebuf.html
```

---

## Вміст

```
твоя_папка/
├── oled_simulator.py   ← симулятор (не змінювати)
├── main.py             ← тут пишеш свій код
└── OLED_1inch5.py      ← драйвер (потрібен тільки на Pico)
```

## Швидкий старт

**1.** Поклади `oled_simulator.py` та `main.py` в одну папку

**2.** Напиши свій код у `main.py`

**3.** Запусти симулятор:
```bash
python oled_simulator.py
```

Відкриється вікно 512×512 пікселів — масштаб 4× від реального дисплея (128×128).

## Як писати код у main.py

Код у `main.py` пишеться **точно так само, як для реального Pico** — жодних змін не потрібно.

```python
from machine import Pin, SoftI2C
from OLED_1inch5 import OLED_1inch5
from time import sleep

# Стандартне підключення
i2c  = SoftI2C(scl=Pin(7), sda=Pin(6), freq=1_000_000)
OLED = OLED_1inch5(0x3d, i2c)

def task1():
    OLED.fill(0)
    OLED.text("Hello World", 10, 60, OLED.white)
    OLED.show()
    sleep(3)

while True:
    task1()
```

Симулятор автоматично підміняє модулі `machine`, `OLED_1inch5` та `time` —  
імпорти залишаються незмінними.

## Правила іменування функцій

Симулятор знаходить функції автоматично за префіксом імені.  
Підтримуються два формати:

| Формат | Приклад |
|---|---|
| `frame_*` | `frame_text()`, `frame_shapes()`, `frame_animation()` |
| `task*` | `task1()`, `task2()`, `task_demo()` |

Функції відображаються у тому порядку, в якому вони визначені у файлі.  
`while True` внизу файлу симулятор ігнорує — він керує циклом сам.

---

## Керування

| Дія | Кнопка | Клавіша |
|---|---|---|
| Наступна функція | `Next ▶` | `→` |
| Попередня функція | `◀ Prev` | `←` |
| Закрити симулятор | — | `Escape` |

Поточна функція відображається у підписі під дисплеєм: `1/3 — task1()`

## Підтримуваний API

Симулятор підтримує повний API класу `OLED_1inch5` / `FrameBuffer`:

### Заповнення
```python
OLED.fill(c)              # залити весь екран кольором c (0–15)
```

### Пікселі
```python
OLED.pixel(x, y, c)       # намалювати піксель
```

### Лінії
```python
OLED.line(x1, y1, x2, y2, c)   # довільна лінія
OLED.hline(x, y, w, c)          # горизонтальна
OLED.vline(x, y, h, c)          # вертикальна
```

### Фігури
```python
OLED.rect(x, y, w, h, c)           # контур прямокутника
OLED.rect(x, y, w, h, c, True)     # заповнений прямокутник
```

### Текст
```python
OLED.text("рядок", x, y, c)   # 8×8 пікселів на символ, ASCII
```

### Відображення
```python
OLED.show()    # передати буфер на екран (обов'язково після кожного малювання!)
```

### Кольори
```python
OLED.white   # = 15  (максимальна яскравість)
OLED.black   # = 0   (вимкнено)
# будь-яке ціле від 0 до 15 — відтінок сірого
```

---

## Відтінки сірого

Реальний дисплей SSD1327 має **4-бітну шкалу сірого** — 16 рівнів яскравості.  
Симулятор точно її відтворює:

```
0   →  вимкнено (чорний)
1–7 →  темно-сірий
8   →  середній сірий
9–14 → світло-сірий
15  →  максимальна яскравість (білий)
```

---

## Переривання sleep() кнопкою

Коли у функції є `sleep(N)` — натискання `Next ▶` або `◀ Prev` одразу  
перериває очікування і переходить до наступної/попередньої функції.  
Це зручно для статичних кадрів, де не хочеться чекати повного таймера.

```python
def task1():
    OLED.fill(0)
    OLED.text("Натисни -> щоб далі", 1, 60, OLED.white)
    OLED.show()
    sleep(60)    # чекає 60 сек або натискання кнопки
```

---

## Обмеження симулятора

| Обмеження | Деталі |
|---|---|
| Тільки ASCII у `text()` | Кирилиця не відображається — так само як на реальному Pico |
| Немає реального I2C | `SoftI2C`, `Pin` — заглушки, нічого не роблять |
| Немає `framebuf.ellipse()` | Метод поки не реалізований |
| `sleep_ms()` | Підтримується (`sleep_ms(100)` = `sleep(0.1)`) |

## Типові помилки

**Екран порожній після запуску**
```python
# Забув OLED.show() !
OLED.fill(0)
OLED.text("test", 0, 0, 15)
# OLED.show()  ← без цього нічого не з'явиться
```

**Симулятор не знаходить функції**
```
[simulator] Не знайдено жодної функції.
  Назви функції як: frame_*()  або  task*()
```
→ Перейменуй свої функції: `def task1():` або `def frame_text():`

**`tkinter` не встановлено (Linux)**
```bash
sudo apt install python3-tk
```

## OLED Pixel Editor

Окремий інструмент для візуального малювання на сітці 128×128 з автоматичною генерацією MicroPython-коду.

### Запуск
```bash
python oled_pixel_editor.py
```

### Інструменти

| Інструмент | Опис | Керування |
|---|---|---|
| Піксель | Один піксель | Клік або зажати і тягнути |
| Лінія | Довільна лінія (алгоритм Брезенхема) | Клік початок → клік кінець |
| Hline | Горизонтальна лінія | Клік початок → клік кінець |
| Vline | Вертикальна лінія | Клік початок → клік кінець |
| Rect | Контурний прямокутник | Клік кут → клік протилежний кут |
| Rect fill | Заповнений прямокутник | Клік кут → клік протилежний кут |
| Заливка | Заливає замкнену область | Клік всередині області |

### Яскравість

Повзунок `c=` від 1 до 15 — відповідає рівням сірого SSD1327.  
Кольоровий квадрат поруч показує реальний колір на дисплеї.

### Збереження коду

- `File → Save code` або `Ctrl+S`
- Створює файл `oled_drawing.py` у тій самій папці

Вміст `oled_drawing.py`:
```python
# Згенеровано OLED Pixel Editor
# Вставте цей код у свою функцію task*() або frame_*()

OLED.fill(0)
OLED.pixel(64, 64, 15)
OLED.line(0, 0, 127, 127, 10)
OLED.rect(10, 10, 50, 30, 8)
OLED.show()
```

### Undo

- `Edit → Undo` або `Ctrl+Z` — скасовує останню дію (будь-яку кількість разів)

### Сітка

На полотні відображається допоміжна сітка кожні 8 пікселів та центральні лінії (x=64, y=64) — для зручного позиціонування елементів.
